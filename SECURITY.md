# Security Policy

## Supported Versions

Only the latest tagged release receives security updates.

## Reporting a Vulnerability

If you discover a security vulnerability in ANANSI CLI, please report it privately by emailing the QYVORA OffSec team. **Do not open a public GitHub issue.**

We will acknowledge receipt within 48 hours and provide an estimated timeline for a fix. Once the fix is released, we will credit the reporter (if desired) in the release notes.

## Scope

This policy covers the ANANSI CLI source code, build pipeline, and release artifacts. Issues related to the targets being scanned (e.g., a vulnerability discovered *by* ANANSI on a target) are out of scope — those should be reported to the target owner.

## Responsible Disclosure

We ask that you give us reasonable time to patch and release a fix before any public disclosure.
